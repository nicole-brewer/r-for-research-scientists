5 Putting It All Together: The Workflow

To contribute to a project, we have to interact with Git, Github, documentation, and our package all in one session.

  1. git pull upstream master
  2. make one incremental change and verify that your code still runs
  3. Add roxygen comments to your .R files
  4. devtools::document() to automatically turn comments into Rd files
  5. devtools::check() and fix any errors given
  6. Repeat devtools::check() until there arenâ€™t any more errors
  7. make a commit with an informative title
  8. git push origin master

To play around with the package from the user standpoint, we must load it with devtools::install()
